====================================================================
         HOW TO RUN / RESTART / DEBUG — Redirection Detector
====================================================================


--------------------------------------------------------------------
1.  STARTING / RESTARTING THE BACKEND  (FastAPI + Uvicorn)
--------------------------------------------------------------------

  Open a NEW terminal (PowerShell / CMD) and run:

      cd  d:\Url\version2\Project\backend
      .\venv\Scripts\activate
      python run.py

  The backend will start on  http://127.0.0.1:8000

  To RESTART:
    • Press  Ctrl + C  in the terminal where backend is running
    • Wait for it to stop
    • Run  python run.py  again

  Quick health check after starting:
    • Open browser → http://127.0.0.1:8000/api/ping
    • You should see  {"status":"ok"}


--------------------------------------------------------------------
2.  STARTING / RESTARTING THE FRONTEND  (Vite + React)
--------------------------------------------------------------------

  Open a SEPARATE terminal (PowerShell / CMD) and run:

      cd  d:\Url\version2\Project\frontend
      npm run dev

  The frontend will start on  http://localhost:5173  (default Vite port)

  To RESTART:
    • Press  Ctrl + C  in the terminal where frontend is running
    • Run  npm run dev  again


--------------------------------------------------------------------
3.  API KEY FILES  (Edit these when you need to change keys)
--------------------------------------------------------------------

  All API keys are stored in ONE file:

      d:\Url\version2\Project\backend\.env

  Contents:
    VIRUSTOTAL_API_KEY   = your VirusTotal API key
    OLLAMA_API_URL       = http://localhost:11434
    OLLAMA_API_KEY       = your Ollama / LLM API key
    OLLAMA_MODEL         = model name (e.g. gpt-oss:120b-cloud)
    DOMAINDUCK_API_KEY   = your DomainDuck API key

  ► To change a key, open the .env file in any text editor,
    replace the value, save, and RESTART the backend.

  Other credential files (Gmail OAuth — usually no need to edit):
    d:\Url\version2\Project\backend\credentials.json   — Google OAuth client config
    d:\Url\version2\Project\backend\token.json          — Auto-generated Gmail token


--------------------------------------------------------------------
4.  VIEWING BACKEND LOGS / SHELL OUTPUT
--------------------------------------------------------------------

  The backend prints all logs directly to the terminal where
  you ran  python run.py

  To see what is happening:
    • Just look at that terminal window
    • All HTTP requests, errors, print() statements, and
      background Gmail scan logs will appear there in real time

  Tip: Keep the backend terminal open and visible while testing.
       You will see lines like:
         INFO:     127.0.0.1:XXXX - "GET /api/ping HTTP/1.1" 200 OK
         Running scheduled Gmail scan...
         Background scan error: ...  (if any)


--------------------------------------------------------------------
5.  QUICK START — BOTH TOGETHER
--------------------------------------------------------------------

  Terminal 1 (Backend):
      cd  d:\Url\version2\Project\backend
      .\venv\Scripts\activate
      python run.py

  Terminal 2 (Frontend):
      cd  d:\Url\version2\Project\frontend
      npm run dev

  Then open  http://localhost:5173  in your browser.
