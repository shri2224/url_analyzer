## 4. Detailed Technical Analysis  

### Malicious Code / Patterns  

```html
<script>
  eval(String.fromCharCode(40, 50, 51, 52));  // ID:50
</script>
```
* `String.fromCharCode(40,50,51,52)` → "(234".  
* The snippet is deliberately simplistic in this sample, but the pattern (eval + fromCharCode) is a known **dropper/obfuscation** method that can reconstruct any JavaScript payload at runtime (e.g., base64‑decoded strings, hex‑packed code).  
* The `eval` execution context gives the attacker **full JavaScript control** over the page, enabling DOM manipulation, additional network requests, or credential stealing.

```html
<iframe src="hxxp://malicious.site" style="display:none"></iframe>
```
* `display:none` makes the iframe invisible to the user – a classic **silent redirect / iframe chain**.  
* The URL is deliberately malformed (`hxxp`) in the example; a real exploit would use `http` or `https`.  
* Hidden iframes are frequently used for:  
  * **C2 download** of secondary payloads (e.g., mining scripts, ransomware).  
  * **Click‑fraud** – loading ad pages in the background to generate impressions/clicks.  
  * **Tracking / data exfiltration** – the remote page can read cookies via `document.cookie` if same‑origin, or can set up a postMessage channel.
